void LinePickingAllProblems ( char **  names,
char **  descriptions 
)

This function is passed two arrays of uninitialised pointers to strings. On its return these pointers are initialised. The firat contains pointers to strings containing the names of all the problems implemented and the second contains pointers to strings containing the descriptions of all the problems implemented.

Parameters
$namePointer to array that this function will fill with pointers to strings containing the names of all the problems currently implemented.
$descriptionPointer to array that this function will fill with pointers to strings containing the descriptons of all the problems currently implemented.
Returns
The two uninitialised arrays passed to the function are initialised with pointers to strings.

Implements the CDF of the distance between two random points for the given problem.

Parameters
$tAn array of distances to calculate the cumulative density for.
$gAn array in which the calculated cumulative densities are returned.
$NThe value pointed to by $N is the number of entries in $T
$problemThe number of the problem for which the PDF will be calculated.
$parametersPointer to the values required to describe the geometry of the problem.
$NparThe number of parameters that $parameters contains.
$resultPointer so that the result of the evaluation can be returned. A non-zero value indicates there was a problem in the input. A description of the error is returned in $error_str. The integer values returned are described in LinePickingCheckParameters.
$error_strA string describing the result of evaluating the function.
Returns
The required calculated cumulative densities are returned in $g.

void LinePickingCheckParameters ( int *  problem,
double *  parameters,
int *  Npar,
int *  result,
char **  error_str 
)

Determines if the supplied parameters are valid input for other functions in this library.

Parameters
$problemThe number of the problem for which the the supplied $parameters are checked against.
$parametersPointer to the values required to describe the geometry of the problem.
$NparThe number of parameters that $parameters contains.
$resultPointer so that the result of the evaluation can be returned. A non-zero value indicates there was a problem in the input. The following values are implemented:
  • 0: parameters are valid.
  • 1: unsupported problem.
  • 2: parameters out of range.
  • 3: not enough parameters were entered.
  • 4: other error..
$error_strA string describing the result of evaluating the function.
Returns
The validity of the supplied parameters as input for other functions in this library is retuned in $result.
